## 🚀 오늘의 주요 작전 (MZ 감성 & UI 고도화)

### 1. 🧢 아이템 레벨 제한 동기화 및 버그 수정
- **현상**: `SHOP_ITEMS` 데이터상으로는 'Red Cap'이 Lv.6(청소년) 제한이었으나, UI(인벤토리 트레이)에서는 Lv.10으로 하드코딩되어 표시되는 현상 발생.
- **해결**: `InventoryTray.tsx` 내 `getCantEquipReason` 함수의 하드코딩된 레벨 체크 로직을 제거하고, `item.requiredLevel` 데이터를 동적으로 참조하도록 수정. 데이터 정합성 100% 확보! 🫡

### 2. ⚡️ PWA 캐시 버스팅 (Cache Busting)
- **현상**: 코드 수정 후 배포해도 사용자 브라우저에 이전 버전의 캐시가 남아있어 변경사항이 즉시 반영되지 않음.
- **해결**: `package.json` 버전을 `0.1.0`으로 범핑하여 서비스 워커(Service Worker)가 새 버전을 인식하고 즉시 업데이트하도록 유도.

### 3. 🖼️ 빌드 시 이미지 깨짐 현상 해결 (Vite Asset Optimization)
- **현상**: Vite 프로덕션 빌드 후 상점 아이템 이미지가 정상적으로 로드되지 않는 문제 확인.
- **원인**: 동적 문자열 경로(`/src/assets/...`) 사용 시 빌드 후 해싱된 경로를 찾지 못함.
- **해결**: `shopItems.ts`에서 이미지들을 **Static Import**(`import img from '...'`) 방식으로 변경. Vite의 에셋 파이프라인을 통과시켜 정확한 해싱 URL 확보.

### 4. 🔒 레벨 제한 경고 UI 고도화 (Refined Feedback)
- **현상**: 레벨 부족 시 피드백이 작고 시인성이 낮았으며, 초기 고도화 버전은 레드 배경이 너무 강렬함.
- **해결 (대표님 피드백 반영!)**: 
    - **세련된 미니멀리즘**: 눈에 피로를 주는 빨강 바탕색을 제거하고, 은은한 블랙 오버레이와 블러 처리로 텍스트 집중도 향상.
    - **가독성 확보**: 멘트 노출 시간을 **3초**로 연장하여 사용자가 충분히 인지하도록 개선.
    - **MZ 감성 유지**: "안돼!"라고 외치는 듯한 **Shake 애니메이션**과 큼지막한 자물쇠(🔒) 아이콘으로 힙한 느낌 극대화.

### 5. 🏠 상점 UI 원복 및 최적화
- **내용**: 고해상도 2D 이미지 적용 후, 대표님의 "심플한 이모지가 MZ 감성에 더 맞다"는 의견을 수렴하여 다시 **이모지 기반 UI**로 원복.
- **효과**: 앱 용량 경량화 및 직관적인 아이콘 시스템 유지.
